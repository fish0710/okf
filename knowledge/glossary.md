# OKF 术语表

| 术语 | 含义 |
| --- | --- |
| bundle | 一组按目录组织、可整体交换的 OKF 文件。 |
| concept | 一个独立知识对象，例如表、指标、API 或运行手册。 |
| frontmatter | Markdown 文件顶部的 YAML 元数据块。 |
| producer | 生成 OKF 文件的人、脚本或 Agent。 |
| consumer | 读取或可视化 OKF 文件的工具或 Agent。 |
| progressive disclosure | 先读入口，再按需进入详细概念的导航方式。 |

概念文件的结构见 [概念文件模型](foundations/concept-document.md)，生产者和消费者的边界见 [生产者与消费者](foundations/producer-consumer.md)。
