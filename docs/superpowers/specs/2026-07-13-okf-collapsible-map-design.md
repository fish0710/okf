# 可折叠知识地图设计

## 目标

让知识地图默认收起，减少单篇文章场景下的视觉占用，同时保留明确、可访问的展开入口。

## 交互设计

- 初始状态为收起，中间地图列缩成窄的纵向开关栏，右侧文章列扩展占据释放出的空间。
- 收起时保留可发现的“展开知识地图”开关；展开后显示标题、图例、关系图和操作提示。
- 点击按钮后展开关系图，再次点击后收起。
- 按钮使用 `aria-expanded` 和 `aria-controls` 表达状态，按钮文字随状态切换为“展开知识地图”或“收起知识地图”。
- 现有概念选择、搜索、筛选和详情行为保持不变。

## 实现边界

- 仅修改 `web/src/main.js` 的面板状态和事件绑定，以及 `web/src/styles.css` 的展开/收起布局。
- 关系图数据模型与 SVG 渲染逻辑不变。
- 增加前端状态测试，覆盖默认收起和展开状态切换。
